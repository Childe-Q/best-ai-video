## 2. 当前阶段的核心判断
- 当前主线应是**站内建设**，不是站外推广。
- 站外不是主线的原因很明确：你已经有足够多的可路由页面，但“可推广页面集合”还不稳定；现在推量，只会把流量送进 fallback-heavy、证据不够硬、SERP 展示不完整的页面。
- 现阶段最适合的策略是**站内为主，站外为辅**。
- 我的建议投入比例：**站内 85%-90%，站外 10%-15%**。
- 站外现在可以做，但只能做“低强度验证型分发”，不能做大规模外链、社媒矩阵、PR 放大或批量 guest post。

## 3. 优先级排序
- **P1：先定义并收口“可推广页面池”**。先把可推广资产限制在一小批成熟页面上，解决“页面很多但成熟度不一”的核心问题；如果不先收口，你后续所有内容、内链、分发都会继续摊薄。
- **P2：补齐核心工具的完整内容包**。优先补 8-10 个核心工具的 `content/pricing/reviews` 三件套，解决落地页质量和转化可信度问题；不先做，工具页看起来会像“模板齐了，正文没补齐”。
- **P3：清理 VS / Alternatives / Feature 的证据与占位内容**。把 placeholder、plan-dependent、`[NEED VERIFICATION]`、弱来源行收掉，解决 SEO 与信任问题；不先做，越推广越放大“像评测站但不够硬”的印象。
- **P4：修正 metadata / schema / 年份 / 内链成熟度规则**。让 canonical、OG、Twitter、schema、date hygiene、ready-only 内链统一，解决抓取与 SERP 展示层面的基础损耗；不先做，站内质量提升也难完整传导到搜索表现。
- **P5：只围绕成熟页面做小规模站外验证**。用少量分发验证哪些页面能拿点击、拿收藏、拿外链，再决定放大；如果现在反过来先做推广，会先暴露站内短板。

## 7. 关于“站内建设 vs 站外推广”的明确结论
- **现在不应该把主要精力放在站外推广上。**
- 原因不是你没有页面，而是“可放心推广的页面比例还不够高”；现在放大，会先放大不一致、证据不稳和页面质量层级差异。
- 如果只做少量推广，我建议占比 **10%**，上限 **15%**。
- 该做的动作：成熟 hub/VS 页的小范围社媒分发、少量人工 outreach、品牌提及、搜索控制台与收录反馈跟踪。
- 不该做的动作：批量外链、PR 放量、群发 guest post、批量目录提交、把所有新页一起往外推。
- 真正可以开始加大站外强度的前提，是核心页面池完成、related links 收口、VS/Alternatives 证据问题明显减少之后。

## 8. 最终建议
- 如果我是负责人，未来一个月我会把主要精力押在三件事：**收口核心页面池、补齐核心工具内容包、把 VS/Alternatives/Feature 的证据和 metadata 质量拉平**。原因很简单：你现在最缺的不是“更多曝光入口”，而是“更稳定、更可信、更可放大的落地资产”。
- 默认选择：未来 30 天不新增大批量新页，不做大规模外链；只允许少量验证型分发，且只推成熟页。等这批页面站内质量稳定，再进入真正的放大阶段。
